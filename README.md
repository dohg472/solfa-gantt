# 솔파 간트

노션 `채널팀 플랜` 데이터베이스를 읽어서 `채널 -> 프로젝트 -> 상세일정` 구조의 간트뷰로 보여주고, 화면에서 드래그하거나 편집한 내용은 개인 페이지 `간트_확인` 안의 `프로젝트` DB에 저장하는 작은 웹앱입니다.

## 실행

```powershell
cd C:\Users\user\Documents\솔파스튜디오\notion-gantt-app
.\start.ps1
```

브라우저에서 `http://localhost:5177`을 열면 됩니다.

로컬 서버가 PC 로그인 후 자동으로 켜지게 하려면 한 번만 아래를 실행합니다.

```bat
install-local-startup-task.cmd
```

이 작업은 로컬 서버만 자동 실행합니다. 노션에 한 번 넣고 계속 쓸 공개 Embed 주소는 아래 `운영용 고정 주소` 설정이 별도로 필요합니다.

## Cloudflare Pages 운영

Render처럼 잠드는 무료 서버를 피하려면 Cloudflare Pages + Functions로 배포합니다. 이 방식은 `*.pages.dev` 무료 고정 주소를 쓰고, 노션 API 키는 Cloudflare 환경변수에 숨깁니다.

Cloudflare 설정 위치와 필요한 값은 [CLOUDFLARE_SETUP.md](./CLOUDFLARE_SETUP.md)에 정리되어 있습니다.

배포 기준:

- 정적 화면: `public`
- API: `functions/api/[[path]].js`
- Embed fallback: `public/_redirects`
- 상태 저장: Cloudflare KV binding `SOLPA_GANTT_KV`
- 노션 원본 자동 동기화: 5분
- 간트에서 저장/수정/숨김: 즉시 Notion `간트_확인` DB에 저장

## 상태 확인

앱이 노션 안에서 안 보이거나, 로컬에서 수정한 내용이 임베드에 안 따라오는 것처럼 보이면 먼저 아래 진단을 실행합니다.

```bat
doctor.cmd
```

진단은 로컬 서버, `채널팀 플랜` 읽기, 원본 보호, `간트_확인` 저장, 삭제 안전장치, 숨김 규칙, 프로젝트 숨김 매칭, 숨김 누수, 숨김 채널 요약, 휴재/휴가성 일정 제외, 제목 접두어 기반 채널 보정, 공유 보기 설정, 간트/목록 경계, 고정 표시 채널, 그룹 기간 조정 손잡이, 안전 병합, 검토 우선 큐/섹션 접기, 날짜축 마커, 고정 Embed 주소, 고정 터널 자동 실행 상태를 한 번에 확인합니다.
`Fixed embed` 또는 `Public URL`이 `NEEDED`이면 아직 운영용 고정 주소가 없는 상태입니다. 이 경우 앱 안의 `임시 주소 켜기`로 테스트용 공개 임베드를 자동 갱신할 수 있지만, PC 재시작 후에도 같은 주소를 보장하려면 고정 주소가 필요합니다.
현재 `.env`의 `APP_AUTO_TEMP_TUNNEL`은 기본적으로 꺼 둡니다. 이 값이 켜져 있으면 로컬 서버 시작 때 테스트용 공개 주소가 자동으로 만들어져 노션 페이지에 불필요한 임베드 블록이 다시 생길 수 있습니다.
앱은 공개 프록시/터널 뒤에서 접속한 hostname을 자동 감지합니다. `gantt.example.com`처럼 고정 공개 hostname이면 Embed 준비 상태로 보지만, `trycloudflare.com` 같은 임시 터널은 공개 주소가 있어도 운영 준비 상태로 보지 않습니다.
운영 패널의 `고정 주소 설정`에서 이미 준비된 공개 주소를 저장하거나 초기화할 수 있습니다. 이 저장은 앱 상태에만 기록되며, 실제 DNS/터널 연결은 `setup-production.cmd -Hostname gantt.example.com` 또는 별도 HTTPS 배포로 준비되어 있어야 합니다.
저장 버튼은 입력한 주소의 `/healthz`가 실제 간트 서버로 응답하는지 확인한 뒤에만 고정 주소로 인정합니다. `localhost`와 `trycloudflare.com` 같은 임시 주소는 운영 고정 주소로 저장되지 않습니다.
`Startup task`가 `CHECK`이면 PC 로그인 후 고정 터널이 자동으로 다시 뜨지 않을 수 있으므로 `setup-production.cmd -Hostname gantt.example.com`으로 고정 주소와 자동 실행을 한 번에 설정합니다.
`Local startup task`가 `OK`이면 최소한 PC 로그인 후 로컬 간트 서버는 다시 켜집니다. 단, 노션 웹/공유용 고정 임베드는 여전히 `APP_PUBLIC_BASE_URL` 또는 `CLOUDFLARE_TUNNEL_HOSTNAME`이 필요합니다.

앱 안의 `운영` 패널에는 `완성 조건` 섹션이 있습니다. 여기서 노션 고정 임베드, 팀플랜 원본 보호, 간트_확인 저장, 공유 보기 동기화, 숨김 규칙, 검토 후보, 삭제 안전장치, 자동 동기화, 일정 리스크를 한 번에 확인합니다. `필수`로 표시된 항목이 남아 있으면 아직 운영형 완성 상태가 아닙니다.
고정 공개 주소가 없을 때는 상단 경고와 운영 액션이 `임베드` 설정을 가장 먼저 보여줍니다. 운영 패널의 상태 목록에는 임시 공개 주소, 노션 임베드 자동 등록, Windows 자동 실행 예약 작업 등록 여부도 같이 표시됩니다. 테스트용 임시는 앱이 노션 블록을 자동 갱신하지만, 영구 운영 주소는 고정 도메인 설정이 필요합니다.

## 노션 연결

1. Notion에서 integration을 만들고 secret을 발급합니다.
2. 대상 데이터베이스 페이지에서 integration에 접근 권한을 줍니다.
3. `.env`의 `NOTION_TOKEN`을 채웁니다. 나머지 값은 현재 `채널팀 플랜`과 `간트_확인 > 프로젝트` 기준으로 이미 매핑되어 있습니다.

```env
NOTION_TOKEN=secret_...
NOTION_READ_DATABASE_ID=20175c38-e3be-4b38-b81e-e8394914431e
NOTION_WRITE_DATABASE_ID=b6e89562-3634-4ef2-bd75-c592a438e372
```

현재 매핑은 아래 흐름입니다.

```env
NOTION_CHANNEL_PROP=채널명
NOTION_PROJECT_PROP=이름
NOTION_DETAIL_PROP=카테고리
NOTION_START_PROP=날짜
NOTION_STATUS_PROP=상태
NOTION_ASSIGNEE_PROP=사람
```

`채널팀 플랜`은 읽기만 합니다. 간트에서 수정하거나 새로 만든 일정은 `간트_확인 > 프로젝트` DB의 `프로젝트`, `채널`, `일정`, `날짜`, `상태`, `담당자`, `메모` 속성으로 저장됩니다.

## 저장 방식

- `채널팀 플랜` 원본 행은 절대 수정하지 않습니다.
- 원본 행을 드래그/편집하면 `간트_확인 > 프로젝트` DB에 사본이 생성되거나 기존 사본이 업데이트됩니다.
- 새 작업은 `간트_확인 > 프로젝트` DB에 생성됩니다.
- 기본 설정에서는 삭제도 원본/대상 DB를 바로 보관 처리하지 않고 화면 숨김으로 처리합니다.
- `연결선`으로 선행/후속 일정을 묶고 `후속` 토글을 켜두면, 선행 일정 이동/종료 변경 시 후속 일정도 같이 이동합니다.
- 후속 일정이 같이 움직일 때는 앱 안의 미리보기 창에서 영향 받는 일정과 날짜 변경을 확인한 뒤 저장합니다.
- `필터` 패널에서 채널, 상세일정, 상태, 담당자, 기간, 업무 기준으로 현재 간트뷰를 좁혀 볼 수 있습니다.
- 빠른 보기로 `진행 중`, `향후 30일`, `촬영`, `업로드`만 바로 볼 수 있습니다.
- 필터, 줌, 밀도, 완료 표시, 연결선, 접기 상태는 브라우저에 자동 저장되어 다시 열어도 유지됩니다.
- 왼쪽 목록의 채널, 프로젝트, 상세일정 이름은 더블클릭해 바로 수정할 수 있습니다.
- 왼쪽 목록의 기간도 더블클릭해 시작일/종료일을 바로 수정할 수 있습니다.
- 상세일정의 상태/담당자 표시도 더블클릭해 바로 수정할 수 있습니다.
- 프로젝트 숨김은 화면 표시명, 원본 프로젝트명, 노션 제목, 에피소드 표기 변형까지 함께 저장해 다시 불러와도 같은 프로젝트가 계속 숨겨지도록 처리합니다.
- 프로젝트를 숨길 때 서버가 현재 `채널팀 플랜`과 `간트_확인` 데이터를 읽어 같은 프로젝트로 매칭되는 일정 ID도 함께 저장하므로, 비슷한 이름의 새 행이 동기화되어도 다시 살아날 가능성을 줄입니다.
- 자동 완료 숨김이나 오래된 검토 후보만 남은 채널은 화면에서 제거하고, 사용자가 직접 숨긴 프로젝트/일정이 있는 채널 또는 고정 표시 채널만 `숨김 유지` 행으로 남깁니다.
- 개별 일정만 숨겨져 같은 프로젝트의 다른 일정이 다시 보이는 경우는 `숨김 누수` 후보로 운영 패널과 `doctor.cmd`에 표시합니다. 이 후보는 `개별 숨김 정리`로 프로젝트 숨김 규칙으로 전환할 수 있습니다.
- `EP.5 + EP.6`처럼 한 원본 행에 여러 에피소드가 함께 적힌 촬영/업로드 일정은 원본 DB를 수정하지 않고 간트 표시용 상세 일정으로 나누어 보여줍니다.
- 임베드 화면은 서버의 데이터 변경뿐 아니라 앱 파일 버전 변경도 감지합니다. 새 버전이 배포되면 편집 중이 아닐 때 같은 Embed 주소에서 자동으로 새로고침됩니다.
- 여러 브라우저/노션 임베드가 동시에 열려도 서버가 30초 동안 노션 조회 결과를 공유합니다. 상단 새로고침 버튼은 이 캐시를 건너뛰고 원본/간트_확인 DB를 다시 읽습니다.
- 채널/프로젝트/관리용 메타 페이지는 원본과 간트_확인 DB에는 그대로 두되, 실제 간트 일정에서는 제외하고 운영 패널의 `메타 제외`로 따로 집계합니다.
- 원본 일정을 간트에서 수정해 `간트_확인` 사본이 생기면 메모 속성에 원본 ID 마커를 남깁니다. 로컬 저장소가 없어도 기존 사본을 다시 찾아 중복 생성을 줄입니다.
- 다중 선택 바와 우클릭 메뉴는 선택된 일정 수, 프로젝트 수, 채널 수, 기간을 함께 보여줍니다. 숨김/삭제 전 적용 범위를 먼저 확인할 수 있습니다.
- 일정/프로젝트/채널을 숨기거나 삭제하기 전에는 앱 안에서 영향 받는 일정 수, 프로젝트 수, 업로드 포함 여부를 먼저 보여주고 확인 후 처리합니다.
- 채널/프로젝트 이름 변경, 선택 일정 일괄 수정, 프로젝트 병합도 저장 전에 변경 전/후 미리보기를 확인합니다.
- 선택한 여러 일정을 `-1일`, `+1일` 이동하거나 연결/연결 해제할 때도 저장 전에 변경 내용을 확인합니다.
- 오른쪽 편집 패널 저장과 우클릭 날짜 변경도 저장 전 변경 항목을 먼저 확인합니다.
- 간트 막대를 드래그하거나 양끝을 늘리고 줄일 때도 저장 전에 날짜 변경을 확인하며, 취소하면 원래 기간으로 돌아갑니다.
- 여러 일정을 선택한 뒤 선택된 막대 하나를 드래그하거나 양끝을 조정하면 선택한 일정 묶음이 함께 이동하거나 기간 조정됩니다.
- 첫 정상 동기화 때 현재 표시 일정을 `자동 운영 기준선`으로 한 번 저장합니다. 운영 패널에서 기준선을 갱신/삭제할 수 있고, 이후 간트 막대 아래에 기준선이 표시되어 일정이 얼마나 밀리거나 당겨졌는지 비교할 수 있습니다.
- 기준선은 앱 서버 상태에 저장되며 `채널팀 플랜` 원본 DB를 수정하지 않습니다.
- 폭이 아주 짧은 채널/프로젝트 막대는 좌우 기간 조정 핸들을 막대 바깥쪽으로 분리하고, 일반 채널/프로젝트 막대도 더 넓은 손잡이를 써서 가운데 이동과 양끝 기간 조정이 겹치지 않게 합니다.
- 채널/프로젝트 이름 변경 확인창은 원본 일정이 간트_확인 사본으로 저장되는지, 기존 간트_확인 일정이 직접 수정되는지 저장 영향을 함께 보여줍니다.
- 새 일정 생성, 숨김 복구, 프로젝트 병합 해제도 실행 전에 앱 안에서 변경 범위를 확인합니다.
- 검토 패널의 `비슷한 이름`, `업로드만 있음`, `업로드 없음`, `완료 숨김 대상` 후보는 `확인` 처리해 목록에서 제외할 수 있고, 숨김 관리에서 다시 해제할 수 있습니다.
- 검토 패널 상단에는 긴급/보통 후보만 추린 `우선 처리 큐`가 있어, 전체 후보를 훑기 전에 바로 처리할 프로젝트부터 확인할 수 있습니다.
- 검토 패널의 빠른 필터는 `전체`, `액션`, `긴급`, `비슷한 이름`, `업로드만`, `업로드 없음`, `완료`, `낮음`을 바로 좁혀 보여줍니다.
- `업로드만`과 `업로드 없음` 후보에는 같은 채널 안에서 날짜가 가까운 반대편 공정 후보를 함께 표시해, 프로젝트명이 달라져 갈라진 촬영/업로드를 더 빨리 확인할 수 있습니다.
- 이름 근거가 강한 근처 후보는 검토 카드의 `근처 병합`으로 바로 프로젝트 병합 기준을 저장할 수 있습니다. 이때도 채널팀 플랜 원본은 수정하지 않습니다.
- 상단 운영 요약은 `긴급 검토` 또는 `검토 액션` 칩을 표시하고, 누르면 검토 패널이 해당 빠른 필터로 열린 상태가 됩니다.
- 상단 운영 요약의 `화면 검토` 칩은 완료 숨김 대상은 유지한 채 현재 간트를 `업로드만`/`업로드 없음` 프로젝트로 바로 좁혀 보여줍니다.
- 상단 운영 요약의 필터 칩은 적용 후 첫 결과 행과 해당 날짜 구간으로 자동 이동합니다.
- 운영 패널의 우선 검토 큐에서도 업로드만 있음/업로드 없음 후보를 바로 숨김 처리할 수 있어, 별도 패널을 열지 않고 당일 정리 항목을 줄일 수 있습니다.
- 검토 패널의 각 후보 섹션은 접기/펼치기를 지원해, 대량 후보는 요약만 보고 필요한 섹션만 펼쳐 확인할 수 있습니다.
- 촬영/편집만 있는 프로젝트와 업로드만 있는 프로젝트가 시기상 이어지고 이름이 닮은 경우는 `비슷한 이름` 후보로 올리며, 간격/유사도 근거를 함께 보여줍니다. 확신도가 낮은 후보는 자동 병합하지 않고 보기/선택/확인만 제공합니다.
- 다른 창이나 노션 임베드는 서버 변경 버전을 최대 5초 안에 감지해 다시 읽고, 창이 다시 보일 때 즉시 동기화합니다.
- 노션 원본 DB 자체의 외부 변경 확인을 위해 전체 새로고침도 주기적으로 수행합니다.
- 최근 편집, 드래그, 숨김, 프로젝트 병합, 새 작업 생성은 상단 `↶` 버튼이나 `Ctrl+Z`로 되돌릴 수 있습니다.
- 왼쪽 목록 폭은 경계선을 드래그해 조정할 수 있고, 조정 값은 공유 보기 설정으로 저장되어 노션 임베드와 localhost 화면에 같이 반영됩니다. 경계선을 더블클릭하면 기본 폭으로 돌아갑니다.
- 날짜축에는 주말 음영과 월 경계선이 표시되어 긴 프로젝트를 볼 때 날짜 구간을 더 쉽게 읽을 수 있습니다.
- 짧은 프로젝트 막대는 이름을 막대 밖에 표시해, 촬영/업로드 상세일정은 색으로 읽고 프로젝트명은 계속 보이도록 했습니다.
- 긴 일정표에서는 `Shift + 마우스 휠`로 좌우 이동하고, `Ctrl/⌘ + 마우스 휠`로 마우스가 가리키는 날짜를 중심으로 확대/축소할 수 있습니다.
- 오른쪽 아래 미니 타임라인 개요에서 전체 프로젝트 분포를 보고, 보이는 구간 손잡이를 드래그하거나 트랙을 클릭해 긴 기간을 빠르게 이동할 수 있습니다.
- 왼쪽 목록이나 검토 패널에서 항목을 선택하면 해당 기간이 화면 밖에 있을 때 타임라인이 자동으로 그 구간을 보여줍니다.
- 다중 선택도 같은 방식으로 동작합니다. 왼쪽 목록의 Shift/Ctrl 선택이나 빈 영역 드래그 선택 후 방금 조작한 기간이 화면 밖에 있으면 타임라인이 자동으로 이동합니다.
- 운영 패널의 `완성 조건`은 현재 상태를 `n/9`로 요약하고, 공개 고정 Embed처럼 운영 전 꼭 해결해야 하는 항목을 `필수`로 따로 표시합니다.
- 운영 패널의 기준선 갱신/삭제 액션은 로컬 화면과 노션 임베드가 같은 비교 기준을 쓰도록 서버 상태를 갱신합니다. 기준선이 비어 있으면 서버가 첫 정상 일정 목록으로 자동 생성합니다.
- 대상 DB에 접근할 수 없을 때만 `data/local-store.json`에 임시 저장합니다.

## 노션 페이지에 넣기

운영 패널에서 `노션에 등록`을 누르면 `간트_확인 > 프로젝트` DB의 부모 페이지에 현재 간트 임베드 블록을 자동으로 추가하거나 기존 블록을 갱신합니다. 원본 `채널팀 플랜` DB는 이 과정에서도 읽기 전용입니다.

수동으로 넣고 싶다면 노션 페이지에서 `/embed` 블록을 만들고 앱 주소를 넣으면 됩니다.

현재 고정 공개 주소가 없으면 로컬 주소가 등록됩니다. 같은 PC의 노션에서는 확인용으로 쓸 수 있지만, 노션 웹/다른 기기에서도 계속 쓰려면 아래 운영용 고정 주소가 필요합니다.

### 테스트용 임시 주소

아직 고정 도메인이 없고 화면 테스트만 필요할 때 씁니다.

앱 안에서는 `운영 > 임베드 > 임시 주소 켜기`를 누르면 됩니다. 이 버튼은 테스트용 공개 주소만 만들며, 노션 페이지에 임베드 블록을 다시 붙이는 작업은 별도의 `노션에 등록` 버튼으로만 수행합니다.
재부팅 때 노션 페이지 아래에 임베드 블록이 다시 생기지 않도록 `.env`의 `APP_AUTO_TEMP_TUNNEL`은 `false`로 둡니다.

```powershell
start-embed.cmd -Temporary
```

`-Temporary`를 붙인 경우에만 `trycloudflare.com` 임시 주소를 만듭니다. 이 주소는 터널을 다시 만들면 바뀌므로 노션 운영 Embed에는 쓰지 않는 편이 안전합니다.
임시 터널 실행 시에는 `간트_확인` 페이지의 기존 임베드 블록을 자동 갱신하지 않습니다. 노션 페이지에 자동 등록이 꼭 필요할 때만 `start-embed.cmd -Temporary -SyncNotionEmbed`처럼 명시적으로 실행합니다.
이제 `start-embed.cmd`만 실행했을 때 고정 주소가 비어 있으면 임시 주소를 자동으로 만들지 않고 설정 안내 후 멈춥니다.
노션에 한 번만 넣고 계속 쓰려면 아래 운영용 고정 주소 방식이 필요합니다.
앱의 `운영` 패널에서도 현재 주소가 임시/로컬인지 고정 주소인지 확인할 수 있습니다.
`운영 > 고정 주소 설정`에는 지금 비어 있는 설정과 다음 실행 단계가 표시됩니다.

### 운영용 고정 주소

노션에 한 번 넣고 계속 쓰려면 Cloudflare named tunnel과 고정 hostname을 사용합니다.

1. Cloudflare에 연결된 도메인의 서브도메인을 정합니다.
   예: `gantt.example.com`
2. 고정 터널을 실행합니다. 도메인을 넘기면 `.env`의 `CLOUDFLARE_TUNNEL_HOSTNAME`과 `APP_PUBLIC_BASE_URL`도 자동으로 채웁니다.
   이미 `노션에 등록`된 임베드 블록이 있으면 이 과정에서 고정 주소로 자동 갱신됩니다.

```powershell
setup-production.cmd -Hostname gantt.example.com
```

PowerShell 실행 정책으로 막히는 PC에서는 `.cmd` 래퍼를 쓰면 됩니다.

```bat
setup-production.cmd -Hostname gantt.example.com
```

또는 `.env`에 아래 값을 직접 채운 뒤 실행해도 됩니다.

```env
CLOUDFLARE_TUNNEL_NAME=solpa-gantt
CLOUDFLARE_TUNNEL_HOSTNAME=gantt.example.com
```

`CLOUDFLARE_TUNNEL_HOSTNAME`만 채워도 앱은 `https://gantt.example.com`을 고정 공개 주소로 인식합니다.
`APP_PUBLIC_BASE_URL`은 Cloudflare가 아닌 별도 HTTPS 배포 주소를 직접 쓰고 싶을 때만 채우면 됩니다.

3. `.env`를 직접 채웠다면 고정 터널을 실행합니다.

```bat
setup-production.cmd -Hostname gantt.example.com
```

처음 실행할 때 Cloudflare 로그인 브라우저가 열릴 수 있습니다. 인증 후 스크립트를 다시 실행하면 고정 Embed 주소를 출력합니다.

4. PC 로그인 시 자동 실행이 필요하면 한 번만 등록합니다.

```powershell
setup-production.cmd -Hostname gantt.example.com
```

등록 여부는 앱의 `운영` 패널과 `doctor.cmd`의 `Startup task` 항목에서 다시 확인할 수 있습니다.

운영용 Embed 주소 형식은 아래처럼 고정됩니다.

```text
https://gantt.example.com/embed/APP_EMBED_KEY값
```

`APP_EMBED_KEY`는 `.env`에 자동 생성되며, 노션 Embed 주소 안에만 들어갑니다.
